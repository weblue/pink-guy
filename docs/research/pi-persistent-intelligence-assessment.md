# `pi-persistent-intelligence` Phase 0 boundary assessment

This assessment uses the upstream characteristics already pinned in Boss Man's
research record. It is not dependency-adoption evidence and does not install or
execute the package.

## Useful contract overlap

- Append-only canonical JSONL with human-readable projections.
- Evidence records, tombstones, patches, review inboxes, and scoped recall.
- Redacted import/export and diagnostics.
- MIT licensing according to the inspected upstream metadata.

## Gaps against the Boss Man authority contract

- Boss Man must remain the only authority for scope capabilities, promotion,
  contest, supersession, deletion, and context receipts.
- Package-owned schemas cannot replace the versioned Boss Man record and event
  schemas or canonical task/evidence tables.
- Session consolidation may call a selected model, while Boss Man requires
  model-less export, recovery, lifecycle mutation, and FTS retrieval.
- Automatic updates or silent compatibility drift are unacceptable for a custody
  dependency; any later adapter must pin and conformance-test an exact version.
- Retrieved repository or generated content cannot acquire instruction authority
  merely because an extension stored or ranked it.

## Phase 0 conclusion

The first-party SQLite/FTS benchmark is the smaller proof and establishes the
conformance suite an adapter would have to pass. `pi-persistent-intelligence`
remains a design reference and possible replaceable adapter. It is not needed for
the FTS baseline, and adopting it now would add a second mutation surface before
its value over the thin first-party layer has been measured.

No vector or semantic experiment is authorized by this assessment. Such an
experiment follows only after the tracked FTS evidence passes and must improve the
benchmark without weakening scope, provenance, offline operation, or rebuilds.
